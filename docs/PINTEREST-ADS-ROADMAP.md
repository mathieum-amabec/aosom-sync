# Pinterest Ads — Launch Roadmap

Where Pinterest stands today and the three steps to go from "feeds exist" to
"ads running with conversion tracking." Based on a read-only code/env audit
(2026-07-18).

## Current state

**Product catalog: ready.** Aosom Sync serves two live, deduped Pinterest
catalog feeds (RSS 2.0 + `g:` namespace, `g:id` = variant SKU = the same
`retailer_id` as the Meta catalog). Non-live products are excluded, each fetch is
recorded in `feed_syncs`, and the feed is CDN-cached ~10 min with on-demand
refresh via `POST /api/revalidate`.

| Feed | URL | Titles |
|---|---|---|
| Pinterest (FR) | `https://aosom-sync.vercel.app/api/feeds/pinterest` | French |
| Pinterest (EN) | `https://aosom-sync.vercel.app/api/feeds/pinterest-en` | English (`custom.title_en`, FR fallback) |

Generated by `buildPinterestFeed()` in `src/lib/feeds/feed.ts`; routes in
`src/app/api/feeds/pinterest{,-en}/route.ts`. See `FEEDS-SETUP.md` and
`PINTEREST-EN-SETUP.md` for catalog-source setup. Per those guides the **FR
source is already connected** in Pinterest Catalogs.

**Tracking + ads tooling: missing.**

- **No `PINTEREST_*` env vars** — no API token, advertiser/ad-account ID, or app
  credentials. There is no programmatic access to Pinterest.
- **No Pinterest Tag / Conversions API (CAPI)** — the storefront has the Meta
  pixel (ScriptTag + custom web pixel) but **nothing for Pinterest**. Without it,
  conversion optimization, ROAS reporting, and retargeting audiences do not work.
- **No Pinterest Ads-API automation** — unlike the Meta setup
  (`scripts/rebuild-dpa-retargeting.mjs`, campaign/ad-set builders), there is no
  Pinterest campaign/ad-group/catalog-ad code.

## The 3 steps to launch

### Step 1 — Account setup (account-side, no code)

1. Create / confirm a **Pinterest Business** account for Ameublo Direct.
2. Create an **ad account** (Ads → set up billing).
3. **Claim the domain** (`ameublodirect.ca`, and `furnishdirect.ca` for EN) so
   pins link back and conversions can be attributed.
4. Confirm the **catalog is ingesting**: Ads → Catalogs → the FR source
   (`/api/feeds/pinterest`) should show *Active / Completed* with a product count
   near the live feed size. Add the EN source (`/api/feeds/pinterest-en`) as a
   second data source (Language English, Country Canada) — see
   `PINTEREST-EN-SETUP.md`.

**Exit criteria:** ad account live, domain claimed, both catalog sources
ingesting with product groups approved.

### Step 2 — Pinterest Tag + Conversions API (net-new code, the biggest lift)

Conversion optimization and retargeting need first-party tracking. This mirrors
the existing Meta pixel work (`docs/meta-custom-web-pixel.js`,
`src/app/api/pixel/`).

1. **Base tag** on the storefront: fire `pintrk('load', <TAG_ID>)` +
   `pintrk('track', 'pagevisit')` (via ScriptTag or the theme, same install path
   as the Meta ScriptTag at `/api/pixel/script`).
2. **Key events**: `pagevisit`, `viewcategory`, `search`, `addtocart`, and
   **`checkout`** with `value`, `currency`, `order_id`, and `line_items`
   (`product_id` = variant SKU, matching the catalog `g:id`).
3. **Purchase on Checkout Extensibility**: as with Meta, the thank-you page
   blocks ScriptTags — send Purchase server-side or from a custom web pixel
   (Settings → Customer events), same constraint documented for Meta in
   `CLAUDE.md`.
4. **Conversions API (CAPI)** for server-side deduped events (a
   `PINTEREST_CONVERSION_TOKEN` env var + `POST` to Pinterest's events endpoint),
   deduplicated with the browser tag via a shared `event_id`.

**Exit criteria:** Pinterest reports `checkout` events with revenue; a 30-day
site-visitor / add-to-cart audience starts populating.

### Step 3 — Campaigns

Once the tag has data and the catalog is approved:

1. **Prospecting** — a catalog/shopping campaign over the product feed
   (Advantage+-style automatic targeting), optimized for `checkout`.
2. **Retargeting** — dynamic retargeting to site-visitor / add-to-cart audiences
   from Step 2 (needs the tag's ~real audience floor first).
3. **(Optional) automation** — if programmatic campaign builds are wanted like
   the Meta setup, add a `scripts/` Pinterest Ads-API builder
   (`PINTEREST_AD_ACCOUNT_ID` + `PINTEREST_ACCESS_TOKEN`). Otherwise create
   campaigns manually in the Pinterest Ads UI.

**Exit criteria:** at least one prospecting + one retargeting campaign live,
optimized to `checkout`, with conversions attributing in the Ads dashboard.

## Summary

| Layer | Status |
|---|---|
| Product feeds (FR + EN) | ✅ live, ready |
| Catalog connected in Pinterest | ✅ FR connected (per docs); add EN |
| Ad account + domain claim | ❌ account-side, Step 1 |
| Pinterest Tag + CAPI | ❌ net-new code, Step 2 |
| Campaigns | ❌ Step 3 |

The feeds are done. Tracking (Tag + CAPI) is the real work; account setup and
campaigns bracket it.
