## Problem Statement

Managing a Shopify dropshipping catalog sourced from Aosom requires daily synchronization of 10,000+ products (prices, stock, images, descriptions), browsing the full Aosom CSV catalog to select products for import, and running an import pipeline that generates bilingual FR/EN content via Claude API before pushing to Shopify. Currently no tool exists for this... the reference project (`aosom-shopify`) handles scraping + generation + import as disconnected CLI scripts. Aosom Sync consolidates this into a single web app with a UI, automated daily sync, and proper change tracking.

## Status Quo

The reference project at `~/.gstack/projects/aosom-shopify/src/` has 5 scripts:
- `scraper.js` — scrapes aosom.ca product pages (not needed, CSV feed has all data)
- `generator.js` — Claude API bilingual content generation (Quebec-tuned, battle-tested)
- `importer.js` — pushes to Shopify with FR primary + EN metafields
- `merger.js` — merges color variants using 2-letter suffix COLOR_MAP with FR names
- `import-all.js` — orchestrates the pipeline

These work but are manual CLI-only, no sync, no UI, no history, no diff engine.

## Constraints

- Deploys to Vercel (serverless, read-only filesystem, 60s function timeout on Pro)
- 2 users, cookie-based password auth (already built in `src/lib/auth.ts` + `src/middleware.ts`)
- Single Shopify store: 27u5y2-kp.myshopify.com
- Quebec market: French primary, English in metafields
- Vercel Cron for daily sync — requires Vercel Pro plan (free tier: 1 cron/day, no timeout extension)
- Aosom CSV is comma-delimited, ~10K rows, ~5s to fetch+parse
- Shopify Admin REST API: 2 requests/second rate limit (leaky bucket). Rate limit handling with Retry-After header already implemented in `src/lib/shopify-client.ts`.
- Prices in CSV are retail (not wholesale). No markup needed — write CSV price directly to Shopify.

## Premises

1. **French primary** — Shopify title/body in FR, English stored in metafields (`custom.title_en`, `custom.body_html_en`, etc). Matches reference project, targets Quebec market. ✅ AGREED
2. **Turso (libSQL)** — SQLite-compatible hosted DB for sync history + import queue. Works on Vercel serverless. Drop-in replacement for current better-sqlite3 code. ✅ AGREED
3. **Direct access token** — Use `SHOPIFY_ACCESS_TOKEN` env var, drop the speculative OAuth client_credentials flow. Proven in reference project. ✅ AGREED
4. **Import queue needs persistence** — In-memory queue dies on cold start. Move to Turso alongside sync history tables. ✅ AGREED
5. **CSV feed is sufficient** — No scraper needed. The Aosom CSV has descriptions, images, specs. ✅ AGREED

## Recommended Approach

**Approach B: Reference-Aligned Rebuild.** The reference project's patterns are proven on a live Shopify store. The marginal effort over Approach A is ~15 min with CC. Specific changes:

### 1. Database: SQLite → Turso
- Replace `better-sqlite3` with `@libsql/client`
- Connection via `TURSO_DATABASE_URL` (must use `https://` scheme for Vercel serverless compatibility, NOT `libsql://`) + `TURSO_AUTH_TOKEN` env vars
- Use `@libsql/client` in HTTP mode (`createClient({ url, authToken })`)
- File: `src/lib/database.ts`
- Schema:
  ```sql
  sync_runs: id TEXT PK, started_at TEXT, completed_at TEXT, status TEXT,
             total_products INT, created INT, updated INT, archived INT,
             errors INT, error_messages TEXT (JSON array)

  sync_logs: id TEXT PK, sync_run_id TEXT FK, timestamp TEXT,
             shopify_product_id TEXT, sku TEXT, action TEXT, field TEXT,
             old_value TEXT, new_value TEXT

  import_jobs: id TEXT PK, group_key TEXT UNIQUE, product_data TEXT (JSON),
               status TEXT ('pending'|'generating'|'reviewing'|'importing'|'done'|'error'),
               content TEXT (JSON: {title_fr, title_en, body_html_fr, body_html_en,
               meta_description_fr, meta_description_en, tags, product_type, vendor}),
               shopify_id TEXT, error TEXT, created_at TEXT, updated_at TEXT
               -- Re-import: UPDATE existing row (UNIQUE on group_key), reset status to 'pending'

  catalog_snapshots: sku TEXT PK, name TEXT, price REAL, qty INT,
                     color TEXT, product_type TEXT, psin TEXT,
                     updated_at TEXT
               -- Refreshed each sync run. Powers the catalog browser stock/price display.

  sync_cursor: id TEXT PK DEFAULT 'current', last_offset INT DEFAULT 0,
               sync_run_id TEXT, updated_at TEXT
               -- For chunked sync across multiple cron invocations
  ```

### 2. Shopify Client: Simplify Auth
- Remove OAuth client_credentials flow
- Use `SHOPIFY_ACCESS_TOKEN` directly (matching reference project)
- File: `src/lib/shopify-client.ts`

### 3. Variant Merger: Adopt COLOR_MAP
- Port the `COLOR_MAP` from reference `merger.js`:
  ```
  BK→Noir, CG→Gris foncé, DR→Brun foncé, GN→Vert, LG→Gris pâle,
  YG→Vert pâle, SR→Argent, CW→Crème, TK→Gris charbon, GY→Gris,
  BU→Bleu, BN→Brun, BG→Beige, DB→Bleu foncé, GG→Vert forêt,
  KK→Kaki, WN→Noyer
  ```
- `parseSku()` algorithm: Check last 2 chars of SKU against COLOR_MAP. If match AND base length >= 3 (sku.length > 4), split into `{base: sku[0..-2], colorCode: suffix, color: frenchName}`. Otherwise, SKU has no color variant (standalone product).
- Strip color from titles BEFORE passing to Claude (not after). Use `stripColorFromTitle()` from reference `merger.js:87-104`: regex removes trailing `, color` or ` - color` or ` – color` patterns. Color word list = all COLOR_MAP French values + common FR/EN color words (noir, vert, gris, black, grey, etc.). This ensures Claude never weaves color into the sentence.
- Variant option name: "Couleur" (not "Color")
- Grouping priority: PSIN field (from CSV, confirmed present: tested with live feed, 10,121 rows, PSIN populated on all rows) > parseSku() fallback
- File: `src/lib/variant-merger.ts`

### 4. Content Generator: Quebec-Tuned Prompt
- System prompt rules (from reference `generator.js`):
  - French must sound natural for Quebec shoppers (not Parisian French)
  - Use metric units (cm, kg), convert if needed
  - Do NOT mention shipping (free or otherwise) — shipping policy is store-level, not per-product
  - Titles under 80 characters
  - HTML body: clean, mobile-friendly
  - 5-8 bullet-point features
  - Meta description: 160 chars max per language
- Output structure: `{title_fr, title_en, body_html_fr, body_html_en, meta_description_fr, meta_description_en, tags}`
- `vendor` is always "Aosom" (hardcoded, not generated by Claude)
- `product_type` comes from CSV `Product_Type` field (not generated by Claude)
- Input to Claude: product name (color-stripped), description (with [BRAND NAME] replaced), specs, material. Do NOT pass color or size info (those are variant-level, not product-level).
- Model: `claude-sonnet-4-6` (balances quality + speed + cost)
- **Timeout/batch strategy:** Content generation happens one product at a time when the user clicks "Generate" in the import UI (single API call, well within 60s). "Generate All" is client-driven: the browser calls `POST /api/import/generate` for each pending job sequentially, updating the UI after each response. This avoids the Vercel 60s function timeout entirely because each call processes exactly one product. The browser shows progress (3/20 generated...) and can be stopped/resumed.
- File: `src/lib/content-generator.ts`

### 5. Shopify Import: FR Primary + Metafields
- Product `title` = `title_fr`, `body_html` = `body_html_fr`
- Metafields stored on the product:
  - `custom.title_en` (single_line_text_field) — English title
  - `custom.body_html_en` (multi_line_text_field) — English description
  - `custom.meta_description_fr` (single_line_text_field) — FR SEO description
  - `custom.meta_description_en` (single_line_text_field) — EN SEO description
  - `custom.source_url` (url) — always empty for CSV-sourced products (no individual page URL in CSV feed)
- Native Shopify SEO: set `metafields_global_description_tag` to `meta_description_fr`
- Import as `draft` status (not active) for manual review
- `inventory_management: null` for dropship — Shopify does not track stock
- Stock from CSV is stored in Turso `sync_logs` for display in the catalog browser only, not pushed to Shopify
- File: `src/lib/shopify-client.ts`

### 6. Import Queue: Persist to Turso
- New `import_jobs` table: id, group_key, product_data (JSON), status, content (JSON), shopify_id, error, created_at, updated_at
- Replace in-memory Map with DB reads/writes
- Survives serverless cold starts
- File: `src/lib/import-pipeline.ts`

### 7. Daily Sync: Scope Clarification
- **The daily sync only updates products already imported to Shopify** (matched by SKU)
- It syncs: price changes and image changes on existing Shopify products
- It does NOT sync stock to Shopify (inventory_management is null for dropship)
- Stock/qty from CSV is written to `catalog_snapshots` table for the catalog browser
- Products removed from CSV: set to `draft` status on Shopify
- New products appearing in CSV: visible in catalog browser, not auto-imported
- Image comparison: sorted URL list equality (same as current `diff-engine.ts`). Full image array replaced on mismatch.
- compareAtPrice on existing products: preserved as-is during sync (not touched)
- **Vercel timeout strategy:** Fetch CSV (~5s) + Shopify products (~3-10s) in parallel. Diff is <1s. Apply phase: 2 req/s rate limit. Safe limit: ~100 changed products per 60s invocation (accounting for Turso writes ~50ms each, cold start ~2s, occasional 429 retries). If more changes than fit, store progress cursor in `sync_cursor` table, resume on next cron invocation. Cursor reset after full completion. Lock: check `sync_runs` for status='running' before starting.
- **Error recovery:** Failed Shopify API calls are logged to `sync_logs` with action='error'. Cursor advances past failures. Failed products are retried on next daily run (they'll still diff as changed).
- File: `src/lib/sync-engine.ts`

## Resolved Questions

1. **Price markup?** No. CSV prices are retail, written directly to Shopify.
2. **compareAtPrice?** Deferred. Not in v1 scope. Can be added later as a pricing rule.
3. **Auto-import new CSV products?** No. New products appear in the catalog browser for manual selection. The import pipeline requires content generation and manual review before pushing to Shopify.

## Success Criteria

- Daily cron sync runs within 60s on Vercel Pro, logs all price/image changes to Turso
- Catalog browser: server-side filtering + pagination (50 products/page via `/api/catalog`). Filters: category tree (from `Product_Type` hierarchy), price range (min/max), stock status (in-stock toggle), text search (name/SKU). Data source: `catalog_snapshots` table (refreshed each sync), NOT re-fetching the CSV on every browser request.

- Color variants merge correctly with French color names ("Noir", "Gris", "Bleu")
- Sync history UI shows audit trail: which products changed, what field, old vs new value
